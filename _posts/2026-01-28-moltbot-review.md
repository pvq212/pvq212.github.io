最近 AI Agent 的話題炒得很熱，我也忍不住跟風嘗試了最近剛改名的 Moltbot（前身是 Clawdbot）。原本以為這會是一個能大幅提升生產力的神兵利器，沒想到在折騰了幾天之後，我得到的只有滿滿的挫折感與憤怒。這篇文章不是什麼推廣文，而是一個受害者的真實告白。如果你正在考慮部署這個工具，請務必先看完這篇評論，再決定要不要把你的系統權限交給它。

## 什麼是 Moltbot

簡單來說，Moltbot 是一個開源的、可自託管的 AI 助手。它的賣點在於「主動性」，也就是它不只是被動回答問題，還能根據設定主動執行任務，並整合了超過 50 種不同的服務。

然而，這個專案最近最出名的事情不是它的功能，而是它的改名風波。因為與 Anthropic 的商標糾紛，原本叫 Clawdbot 的它在 2026 年 1 月被迫改名為 Moltbot。這種品牌混亂只是冰山一角，背後反映的是整個專案開發節奏的混亂與不成熟。

## 令人崩潰的安裝過程

如果你以為這是一個下載即用的軟體，那你就大錯特錯了。Moltbot 的安裝過程極其繁瑣，你需要處理 Docker 容器、設定複雜的環境變數，還要手動配置各種 API Key。對於一般使用者來說，這門檻高得離譜；而對於開發者來說，這種過度設計的架構也讓人感到疲憊。我花了一個下午才讓它勉強跑起來，結果換來的卻是接下來一連串的災難。

## 我的負面體驗：慢、貴、沒用

首先要吐槽的就是回應速度。在現在這個追求即時反饋的時代，Moltbot 的反應遲鈍得令人髮指。同樣的問題，我直接去問 ChatGPT 或 Claude 可能三秒就有答案，但透過 Moltbot，它會先在那邊「思考」半天，然後再慢吞吞地吐出結果。這種效率，真的好意思自稱是生產力工具嗎？

更糟糕的是它的 Token 消耗機制。Moltbot 有一個所謂的 Heartbeat 機制，用來維持它的主動監控功能。這意味著即使你根本沒在跟它說話，它也在背景默默地消耗你的 API Credits。我才放著跑兩天，帳單就莫名其妙多了一截。這種設計簡直是在偷錢，對於預算有限的個人使用者來說，這根本是個錢坑。

而且，它給人的感覺更像是一個昂貴的玩具，而不是正經的工具。雖然號稱有很多整合功能，但實際用起來卻到處是 Bug。設定半天，最後發現它連一個簡單的行事曆同步都做不好，這種挫敗感真的會讓人想直接刪除整個 Docker Image。

## 權限過大與安全疑慮

這是我最想警告大家的一點。Moltbot 要求的權限大得驚人，它需要 Shell Access、Filesystem Access，甚至還要控制你的 Browser。

{: .prompt-warning }
給予一個尚不成熟的開源專案如此高的系統權限是非常危險的行為。一旦程式碼存在漏洞，你的整個伺服器甚至個人電腦就等於對外敞開大門。

事實證明我的擔憂並非空穴來風。就在 2026 年 1 月，Moltbot 爆發了嚴重的 Dashboard 洩漏事件。因為預設配置的安全性不足，數百個公開在網路上的 Moltbot 實例被發現可以直接存取，導致使用者的 API Keys 大規模暴露。這種低級的安全錯誤發生在一個要求高權限的工具上，簡直是不可原諒的。

## 總結：請遠離這個工具

總結來說，Moltbot 是一個過度包裝、效率低下且存在嚴重安全隱患的產品。它試圖做很多事情，但沒有一件做得好。

*   **權限過大**：隨意索取系統核心權限，風險極高。
*   **反應遲鈍**：速度完全跟不上主流 AI 工具。
*   **成本高昂**：Heartbeat 機制讓你的 API 帳單失控。
*   **品牌混亂**：改名風波顯示了團隊的應對能力不足。
*   **安全災難**：Dashboard 洩漏事件已經證明了它的不安全性。

如果你只是想玩玩 AI Agent，或許可以找台沒用的舊電腦試試，但千萬不要把它當作你的主力生產力工具，更不要在存有重要資料的環境中運行它。對我來說，Moltbot 是一個徹底讓我失望的失敗作，我不推薦任何人使用它。
